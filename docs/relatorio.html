2. Objetivo do Projeto

O sistema tem como objetivo monitorar uma missao espacial experimental, interpretar dados de telemetria, identificar situacoes criticas, gerar alertas automaticos, realizar previsoes simples e recomendar acoes para manter a operacao segura.

3. Dados de Entrada

Os dados sao lidos a partir do arquivo data/telemetria.json, contendo leituras horárias da missao. Os dados foram gerados randomicamente, simulando um ambiente de uma missão espacial, com frequência de coleta das informações a cada hora.

Grupo de dados Exemplos de campos Finalidade
Modulos criticos suporte_vida, energia, comunicacao, habitat, laboratorio, armazenamento Verificar se sistemas essenciais estao operacionais.
Energia geracao_kwh, consumo_kwh, bateria_percentual Avaliar risco energetico e alimentar a previsao.
Ambiente temperatura_c, radiacao_msv, qualidade_comunicacao, velocidade_vento_kmh Identificar riscos ambientais e operacionais.
Eventos tipo, descricao, detalhes, acao_recomendada Registrar ocorrencias relevantes da missao.

4. Estruturas de Dados Utilizadas

Estrutura Uso no sistema
Lista Armazena consumo energetico, geracao energetica e temperaturas.
Fila Armazena alertas pendentes e processa por ordem de chegada.
Pilha Guarda eventos criticos recentes para analise rapida.
Dicionario Permite consultar rapidamente o status dos modulos pelo nome.
Hierarquia / Arvore Representa subsistemas da missao, como energia, habitat e pesquisa.
Matriz Representa leituras energeticas no formato [geracao, consumo, bateria].

5. Diagnostico Operacional

A classificacao da missao pode ser NORMAL, ALERTA ou CRITICO, conforme as condicoes dos modulos, bateria, comunicacao e radiacao.

Expressao booleana principal

CRITICO =
(bateria < 20 AND comunicacao == 0)
OR
(suporte_vida == 0)
OR
(radiacao > 90)
